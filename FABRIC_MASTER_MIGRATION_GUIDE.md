# 🔄 Fabric Master Migration Guide

## 1. Pre-Migration Checklist
- [ ] **Backup**: Export all current fabric data to CSV/Excel as a safety net.
- [ ] **Verify Schema**: Ensure the `base_fabrics` table has the new columns (`base_code`, `construction_code`, `weight`, etc.).
- [ ] **Test Environment**: Run the migration on a staging database first if available.

## 2. Migration Steps (Manual/Scripted)
If moving from an old system or spreadsheet:
1.  **Format Data**: Ensure your source Excel sheet matches the columns expected by the `FabricService.createFabric` method.
2.  **Map Fields**:
    *   Old `Material` -> New `Base`
    *   Old `Type` -> New `Construction`
    *   Old `Code` -> New `SKU` (or let system regenerate)
3.  **Run Import**: Use the Bulk Import feature (if implemented) or a SQL script.

## 3. Post-Migration Verification
1.  **Spot Check**: Randomly select 5-10 fabrics and verify their details in the new UI.
2.  **SKU Check**: Verify that SKUs generated by the new system match expectations.
3.  **Count Check**: Ensure the total number of records matches the source.

## 4. Rollback Procedure
If data is corrupted:
1.  Delete the newly inserted records using the `created_at` timestamp.
